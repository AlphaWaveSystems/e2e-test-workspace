Project structure:
- backend/ (Node.js/Express)
- frontend/ (React)
- mobile/ (React Native)
- reports/ (CI/CD artifacts)

Standards:
- Commit messages: imperative, concise (e.g., "Add user login API")
- Branch naming: feature-<component>-<action> (e.g., feature-auth-login)
- Pull request titles: match commit messages
- No direct main/master merges
